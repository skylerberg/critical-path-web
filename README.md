# critical-path-web

Critical Path — a project-management suite (kanban boards, task dependency graphs,
critical-path highlighting). Svelte 5 SPA + PWA built with Vite; no SvelteKit.

The companion API lives in [`critical-path-api`](../critical-path-api).

## Setup

Requires Node >= 22.

```sh
npm install
```

`npm install` also activates the repo's git hooks (`.githooks/`) via the `prepare` script.
If your npm config sets `ignore-scripts=true`, run `npm run prepare` once manually.

## Development

Start the API first (port 3001, see the `critical-path-api` README), then:

```sh
npm run dev
```

Vite serves the app on <http://localhost:5173> and proxies `/api` to
`http://localhost:3001`.

## Generated API types

`src/api/api.generated.ts` is generated from the API's OpenAPI spec and committed:

```sh
npm run generate:api                       # from a running API (SPEC_URL, default localhost:3001)
SPEC_PATH=../critical-path-api/openapi.json npm run generate:api   # from a dumped spec (npm run openapi:dump in the API repo)
```

Regenerate after any API schema change and commit the result.

## Checks

```sh
npm run check         # svelte-check (type checking)
npm run lint          # eslint
npm run format        # prettier --write
npm run format:check
npm test              # vitest (jsdom)
npm run build         # production build incl. PWA assets
npm run preview       # serve the production build
```

## Production

Serve `dist/` on the same origin as the API with SPA history fallback: non-`/api`
paths that miss static files must rewrite to `/index.html`
(nginx: `try_files $uri /index.html;`), otherwise deep links 404 on refresh.

## Icons

PWA icons in `public/` are generated by `node scripts/generate-icons.mjs` and committed.
